# 청주 지웰시티 부동산 전문 사이트 PRD

> **Claude Code에게**: 이 문서는 청주 지웰시티(SK하이닉스 청주캠퍼스 + 현대백화점 충청점 인접) 지역을 전문으로 하는 공인중개사의 개인 웹사이트 개발 명세입니다. 끝까지 읽고, 불명확한 부분은 질문한 뒤 작업을 시작해주세요.

---

## 1. 프로젝트 개요

### 1.1 배경

- **운영자**: 청주 지웰시티 1·2·3차 블럭을 영업 구역으로 하는 공인중개사 (웹개발자 출신)
- **시장 상황**: 네이버부동산, 직방, 다방, 호갱노노 같은 대형 플랫폼이 매물 리스팅 시장을 장악
- **경쟁자**: 같은 블럭에서 14년차 베테랑 중개사들이 활동 중 ("지웰탑부동산", "지웰동인부동산" 등)
- **차별화 포인트**: 웹개발 역량을 활용한 **데이터 기반 의사결정 도구** + **SK하이닉스 직원 타겟 콘텐츠**

### 1.2 사이트 목적

이 사이트는 **매물 리스팅 사이트가 아닙니다**. 기존 플랫폼과 같은 걸 만들면 무조건 집니다.

대신 다음 가치를 제공합니다:

1. **의사결정 도구**: 지웰시티 1·2·3차 비교, 평형/층/향별 분석, 시세 추이 시각화
2. **타겟 콘텐츠 허브**: SK하이닉스 청주 발령자, 청주 갈아타기 수요층을 위한 가이드
3. **신뢰 자산**: 운영자의 전문성을 보여주는 포트폴리오 + 블로그
4. **리드 수집**: 도구·콘텐츠로 유입된 사용자를 카카오톡/문의 폼으로 연결

### 1.3 타겟 사용자

| 그룹 | 비중 | 주요 관심사 |
|------|------|-----------|
| SK하이닉스 청주캠퍼스 임직원 (30~40대) | 40% | 출퇴근, 전세/월세, 직급별 적정 평형 |
| 청주 자산가 (갈아타기 수요) | 35% | 1차 vs 2차 vs 3차 비교, 시세 전망 |
| 학부모 (서촌초/솔밭초/복대중 학군) | 15% | 학군 배정, 동별 학교 거리 |
| 외부 투자자 | 10% | 갭, 전세가율, 시세 추이 |

## 2. 기술 스택

### 2.1 운영자 보유 스택
- 현업: Spring Boot, Vue.js, MariaDB

### 2.2 본 프로젝트 권장 스택 (변경 사유 포함)

**Spring Boot + Vue.js 대신 Next.js로 가는 이유**:
- Vercel 무료 배포로 인프라 비용 0원
- 클로드코드와 함께 Next.js 풀스택 개발 시 속도가 압도적
- 부동산 사이트엔 무거운 비즈니스 로직이 없어 Spring Boot 오버킬
- Vue 익숙하시면 React/Next.js 적응 1주일이면 충분

**최종 스택**:

| 영역 | 기술 | 비고 |
|------|------|------|
| 프레임워크 | Next.js 15 (App Router) | TypeScript |
| 스타일링 | Tailwind CSS | shadcn/ui 컴포넌트 활용 |
| 차트 | Recharts | 실거래가 시각화 |
| 콘텐츠 | MDX (블로그용) | 인터랙티브 위젯 임베드 가능 |
| 배포 | Vercel | 무료 플랜 |
| 폼/문의 | Tally 임베드 또는 카카오톡 채널 | DB 없이 시작 |
| 분석 | Vercel Analytics + GA4 | |
| (Phase 2) DB | Supabase (PostgreSQL) | 매물 50개 초과 시 도입 |

### 2.3 단계별 DB 전략

**Phase 1 (MVP, ~2개월)**: DB 없음
- 매물: `/content/listings/*.mdx` 파일로 관리 (Git 커밋 → 자동 배포)
- 블로그: `/content/posts/*.mdx`
- 실거래가: 빌드 타임에 국토부 API 호출 → ISR(Incremental Static Regeneration)로 정적 생성, 1일 1회 재생성
- 문의: Tally 폼 또는 카카오톡 채널 링크

**Phase 2 (3개월차~)**: Supabase 도입
- 매물 CRUD 어드민 페이지
- 실거래가 일별 누적 (cron으로 자동 수집)
- 매물 알림 구독자 명단
- 방문자 행동 로그

## 4. 핵심 페이지/기능 명세

### 4.1 홈 (`/`)

**목적**: 첫 방문자에게 "이 사이트는 뭐가 다른가"를 3초 안에 전달

**구성 (위에서 아래)**:
1. **Hero**: 운영자 사진 + "지웰시티 전문, 데이터로 증명합니다" 같은 카피
2. **3대 도구 카드**: 비교 대시보드 / 실거래가 추이 / 하이닉스 매칭 (각 카드 클릭 시 해당 도구로)
3. **최신 매물 3~6개** (큐레이션)
4. **최신 블로그 글 3개**
5. **CTA**: 카카오톡 문의 + 전화

### 4.2 ★ 도구: 지웰시티 1·2·3차 비교 (`/tools/compare`)

**목적**: "1차 살까 2차 살까 3차 살까"를 정량적으로 답해주는 사이트의 시그니처 도구

**대상 단지**:
- 신영지웰시티 1차 (복대동, 2010년 입주, 2,164세대, 주상복합 최고 57층)
- 두산위브지웰시티 2차 (복대동, 2015년 입주, 1,956세대)
- 청주지웰시티 푸르지오 3차 (복대동, 신축)

**보여줄 비교 항목**:

| 카테고리 | 항목 |
|---------|------|
| 시세 | 평형별 매매 평균가, 전세 평균가, 전세가율, 매매-전세 갭 |
| 실거래가 추이 | 최근 3년 평형별 라인 차트 (3단지 동시 표시) |
| 단지 정보 | 입주연도, 세대수, 동수, 용적률, 건폐율, 주차대수 |
| 학군 | 배정 초·중학교, 학교까지 도보 거리 |
| 편의시설 | 지웰시티몰 접근성, 현대백화점 접근성 (도보 분) |
| 교통 | SK하이닉스 청주캠퍼스까지 차량 시간, 서청주IC까지 시간 |
| 관리비 | 평형별 평균 관리비 |

**UX**:
- 상단에 단지 토글 (3차 모두 보기 / 2개만 비교)
- 평형 선택 (39평/49평 등) → 모든 차트가 해당 평형 기준으로 갱신
- 결과를 PNG/PDF로 다운로드하는 "공유하기" 버튼 (입소문용)

### 4.3 ★ 도구: SK하이닉스 직원 매물 매칭 (`/tools/hynix-matcher`)

**목적**: 청주 발령받은 하이닉스 직원이 "어디 살아야 하지" 검색했을 때 답이 되는 도구

**입력**:
1. 직급/연차 (신입 / 대리 / 과장 / 차장 / 부장+)
2. 가족 구성 (1인 / 신혼부부 / 자녀 1명 / 자녀 2명+)
3. 거래 유형 (월세 / 전세 / 매매)
4. 예산 (슬라이더)
5. 우선순위 (체크박스: 출퇴근 시간 / 학군 / 백화점 접근성 / 신축 여부)

**출력**:
- 추천 단지 + 평형 (점수 순 정렬)
- 각 추천에 대한 근거 ("월세 예산 내 가능한 39평 매물 N개", "출퇴근 8분", "솔밭초 배정")
- 해당 조건의 실제 매물로 연결되는 링크
- "운영자에게 상담 요청" CTA

**구현 노트**: 룰 기반 점수 매기기로 충분 (AI 안 써도 됨). 단지별 메타데이터를 JSON으로 두고 입력 조건과 매칭하는 스코어링 함수.

### 4.4 도구: 실거래가 추이 (`/tools/price-history`)

- 단지 선택 → 평형 선택 → 차트
- 데이터 소스: 국토교통부 실거래가 공개시스템 OpenAPI (https://www.data.go.kr/data/15126474/openapi.do)
- 빌드 타임에 호출, ISR로 1일 1회 재생성
- Recharts로 라인 + 거래량 막대 동시 표시
- 평균/중위값 토글

### 4.5 도구: 전세 vs 월세 vs 매매 시뮬레이터 (`/tools/rent-vs-buy`)

- 입력: 자본금, 대출 가능 금액, 보유 기간, 예상 시세 상승률, 대출 금리
- 출력: 3가지 시나리오의 N년 후 순자산 비교 차트
- "내 조건에 가장 유리한 선택은?" 결론

### 4.6 매물 페이지 (`/listings`, `/listings/[slug]`)

**철학**: 매물 수로 승부하지 않음. 큐레이션된 매물 5~20개를 **깊이 있게** 보여줌.

**개별 매물 페이지 구성**:
- 사진 갤러리
- 핵심 정보 (가격, 평형, 층, 향, 입주가능일)
- 운영자의 한 줄 코멘트 ("이 매물의 강점/약점을 솔직하게")
- 단지 정보 자동 임베드 (위 비교 도구 데이터 재활용)
- 동·호수별 조망 시뮬레이션 (가능하면)
- 비슷한 조건의 최근 실거래가 비교
- 문의 CTA

### 4.7 블로그 (`/blog`)

**초기 12편 콘텐츠 캘린더 (Claude Code가 사이트 만들고 나서 채울 글들)**:

1. 지웰시티 1차 vs 2차 vs 3차 완벽 비교
2. SK하이닉스 청주 발령자가 알아야 할 5가지
3. 지웰시티 학군 완벽 정리 (서촌초/솔밭초/복대중)
4. 지웰시티 평형별 추천 가이드 (1인/신혼/4인 가족)
5. 복대동 vs 가경동 vs 율량동 비교 (청주 갈아타기)
6. 지웰시티몰·현대백화점 200% 활용법 (라이프스타일)
7. 전세사기 피하기 - 지웰시티 전세 계약 체크리스트
8. 지웰시티 갭투자, 지금 들어가도 될까 (데이터 분석)
9. SK하이닉스 셔틀버스 정류장과 지웰시티 동선
10. 지웰시티 인테리어 트렌드 (실제 거래 매물 사례)
11. 청주 부동산 시장 분기 리포트 (정기 발행)
12. 거래 후기: 하이닉스 부장님의 39평 매매 사례

**기술 요구사항**:
- MDX 사용 (글 안에 React 컴포넌트 임베드 가능 → 글 안에 차트, 시뮬레이터 박을 수 있음)
- 메타데이터: title, description, date, tags, ogImage
- 자동 생성: 사이트맵, RSS, OG 이미지
- 카테고리: 단지분석 / 하이닉스 / 학군 / 시장분석 / 거래후기 / 가이드

### 4.8 운영자 소개 (`/about`)

- 얼굴 사진 (필수, 신뢰의 핵심)
- 경력, 자격증, 영업 구역
- 거래 완료 사례 (개인정보 가린 후기)
- 운영 철학 ("왜 이 사이트를 만들었나")
- 카카오톡 / 전화 / 사무실 위치 (네이버 지도 임베드)

### 4.9 문의 (`/contact`)

- Tally 폼 임베드 (이름, 연락처, 관심 매물/단지, 문의 내용)
- 카카오톡 채널 큰 버튼
- 전화번호 (모바일에선 tap-to-call)
- 사무실 주소 + 지도 + 방문 가능 시간

## 5. 디자인 가이드

### 5.1 톤앤매너
- **신뢰감 + 전문성**: 흔한 부동산 사이트 같은 화려한 그라데이션 X
- **데이터 친화적**: 차트와 숫자가 메인 콘텐츠. 가독성 최우선
- **모바일 퍼스트**: 트래픽의 70% 이상은 모바일

### 5.2 컬러 (제안, 클로드코드가 더 좋은 안 있으면 제안)
- Primary: 차분한 네이비 또는 딥 그린 (신뢰)
- Accent: 따뜻한 골드 또는 코랄 (CTA용)
- Neutral: 화이트 + 다양한 그레이 톤
- 다크모드 지원

### 5.3 타이포그래피
- 본문: Pretendard (한글 가독성 표준)
- 숫자/데이터: 고정폭 폰트 (JetBrains Mono 등)

### 5.4 참고할 만한 사이트 톤
- Linear, Vercel, Stripe 같은 SaaS 톤 (단정함)
- 부동산 사이트 중에선 호갱노노보다 데이터 시각화에 더 신경 쓴 느낌

## 6. SEO 전략

### 6.1 타겟 키워드

**1순위 (전환율 높음, 경쟁 약함)**:
- "하이닉스 청주 전세", "하이닉스 청주캠퍼스 근처 아파트"
- "지웰시티 1차 2차 차이", "지웰시티 비교"
- "솔밭초 배정 아파트", "서촌초 배정 단지"

**2순위 (검색량 높음)**:
- "청주 지웰시티 시세", "복대동 아파트"
- "청주 갈아타기", "복대동 vs 가경동"

### 6.2 기술적 SEO
- Next.js의 generateMetadata로 페이지별 메타 태그
- JSON-LD 구조화 데이터 (Article, RealEstateListing, LocalBusiness)
- 자동 사이트맵, robots.txt
- OG 이미지 자동 생성 (@vercel/og)
- Core Web Vitals 90점 이상 목표

### 6.3 네이버 SEO 추가 작업
- 네이버 서치어드바이저 등록
- 네이버 블로그 운영 병행 (네이버는 자기 플랫폼 우대) → 사이트 글 요약본을 네이버 블로그에 올리고 본문 링크

## 10. Claude Code에게 전달하는 작업 지시

### 10.1 시작 방법
1. 이 문서를 끝까지 읽고 이해 안 가는 부분, 결정 필요한 부분을 먼저 질문해주세요
2. Sprint 1부터 순차 진행. 한 Sprint 끝나면 사용자에게 리뷰 요청
3. 모든 코드는 TypeScript + 명확한 타입
4. 모든 페이지는 모바일 우선으로 디자인

### 10.2 우선순위 원칙
- **속도 > 완벽**: MVP를 빨리 띄우고 반복 개선
- **콘텐츠 > 기능**: 화려한 기능보다 SEO 잘 잡히는 콘텐츠 구조가 우선
- **데이터 > 디자인**: 차트와 숫자가 정확한 게 첫째, 예쁜 건 그 다음
- **재사용 > 추상화**: 단지가 3개뿐이라 과도한 추상화 불필요

### 10.3 결정이 필요할 때 사용자에게 물어볼 것
- 도메인 이름 (Daerim.com)
- 운영자의 정확한 이름, 사무실 위치, 자격증, 경력
- 이름 : 이명숙
- 사무실 위치 : 충북 청주시 흥덕구 대농로 45 근린상가102호
- 자격증 : 공인중개사
- 경력 : 11년
- 컬러 톤 최종 결정
- 매물 정보 입력 방식 (Markdown 직접 편집 vs 간단한 어드민 툴)

### 10.4 하지 말아야 할 것
- 불필요한 회원가입/로그인 시스템 (Phase 1에선 필요 없음)
- 결제 기능
- 매물을 수백 개씩 가정한 무거운 검색/필터 시스템
- 화려한 애니메이션과 패럴랙스 (모바일 성능 저하)
- 외부 부동산 플랫폼 데이터 크롤링 (법적 리스크)
